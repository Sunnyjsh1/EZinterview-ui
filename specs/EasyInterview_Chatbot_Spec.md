## 1. 프롬프트 구성

| # | 파일 | 역할 |
|---|---|---|
| 1 | `tools/prompts/result_chat_system.txt` | 시스템 프롬프트 — 분석 원칙, 데이터 근거 의무, 인용 규칙, 세그먼트 분석, 컨텍스트 활용 |
| 2 | `tools/prompts/result_chat_presets.json` | 해시태그 5개 프리셋 프롬프트 + 컨텍스트 템플릿 |
| 3 | `tools/prompts/result_chat_output_guide.txt` | 응답 포맷 가이드 + few-shot 예시 5종 (미니 보고서 형식) |

---

## 2. API 호출 구조

```
[system]
  result_chat_system.txt + result_chat_output_guide.txt (합쳐서 전달)

[user - 첫 메시지]
  [자동 첨부 컨텍스트]
  현재 탭: R1/R2/R3
  현재 필터: [{field: "성별", values: ["여성"]}, ...]
  응답자: P01(카테고리), P02(카테고리), ...

  --- 전체 인터뷰 데이터 ---
  {transcript}

  {사용자 입력 또는 프리셋 프롬프트}

[user - 후속 메시지 (멀티턴)]
  {사용자 입력만 — 컨텍스트 재전송 불필요}
```

## 3. 해시태그 프리셋

| 해시태그 | 프롬프트 요약 |
|---|---|
| 경쟁사 비교 분석 | 언급된 경쟁 브랜드 추출 + 자사 vs 경쟁사 인식 비교 |
| 세그먼트별 차이 분석 | 스크리너 변수 기준 응답 패턴 차이 |
| 특정 질문 심층분석 | 현재 선택된 질문({current_question})의 전체 응답 심층 분석 |
| 구매 의향 드라이버 분석 | 긍정/부정 드라이버 구분 + 언급 빈도 + 대표 발화 |
| 핵심 VOC 요약 | 비즈니스 의사결정에 중요한 원문 발화 TOP 5 |

## 4. 응답 포맷 (미니 보고서)

```markdown
### 핵심 발견
- (인사이트 1~2문장, 최대 3개)

### 근거
> "응답자 발화 원문..." — P번호
> (최대 3개)

### 시사점
(실무적 함의 1~2문장)

---
💡 (후속 분석 제안 1문장)
```

**제약:** 전체 1,200자 이내 / 표는 3열 5행 이하 / 인용 최대 3개 / 서론 없이 핵심 발견부터

## 5. 개발팀 구현 사항

### 프론트엔드

| 항목 | 내용 |
|---|---|
| 챗봇 UI | 플로팅 패널 (400x520px), 말풍선 캐릭터 토글 |
| 컨텍스트 수집 | 현재 탭/필터/질문 상태를 첫 메시지에 자동 첨부 |
| Markdown 렌더링 | `**굵게**` → `<strong>`, `> 인용` → 좌측 보더, `- 불릿` → 리스트, `### 제목` → 14px bold |
| 대화 이력 | 세션 내 멀티턴 유지 (첫 메시지에만 컨텍스트 포함) |
| 해시태그 칩 | 클릭 시 프리셋 프롬프트를 user 메시지로 전송 |

### 백엔드

| 항목 | 내용 |
|---|---|
| API 엔드포인트 | `/api/chat` — 프롬프트 조립 + LLM 호출 |
| 시스템 프롬프트 | `result_chat_system.txt` + `result_chat_output_guide.txt` 합산 |
| 데이터 주입 | 전체 STT 텍스트 + 스크리너 변수를 첫 user 메시지에 삽입 |
| 스트리밍 | SSE 또는 WebSocket으로 응답 스트리밍 권장 |
